# 1.サービス名
**BASE LOG（ベースログ）**
野球の試合進行に係るタスク増加、人手不足を解決するサービス。
審判による判定（ボール、ストライク、アウト等）をリアルタイムで連携し、
スコアボードやアプリに同期したり、主審と塁審のコミュニケーションを円滑にすることを目指します。

## 3.内容
- チーム管理機能 (team_admin.php): メンバーの基本情報、スキル、役割などを一元管理
- オーダー登録 (order_kanban.php): 登録したメンバーをカード形式で表示。ドラッグ＆ドロップでスターティングメンバーに登録できる。
- オーダー登録時に、大会IDを入力して試合進行/スコアなどとの連携を想定。
- また過去のオーダー表をコピーして登録できるため、手書きよりも利便性が高い。


#① 判定の入力（主審）
- 主審はPCのテンキー、またはスマホ画面上のスレートグレーの「＋」ボタンを使って判定を入力します。
    - [8]：Ball（ボール）
    - [9]：Strike（ストライク）
    - [5]：Out（アウト）
    - [3]：Foul（ファウル）

#② リアルタイム・フィードバック
- 判定が入力されると、主審・塁審それぞれのスマホに即座にフィードバックが飛びます。
    - 視覚: 判定内容に合わせて画面下部がカラーフラッシュ。
    - 触覚: 強力な振動（バイブレーション）による通知。

#③ メッセージ表示
- フィードバックと同時に、画面中央に判定結果のメッセージが表示されます。
    - 主審画面: 「[判定結果] 操作を取り消しますか？」
    - 塁審画面: 「[判定結果] 判定リクエストを送りますか？」
    ※ 放置または操作継続で自動的にダッシュボード（待機中）へ戻ります。

#④ 操作の取り消し（主審がYesを押した場合）
- 誤入力があった際、主審が「取り消し」ボタン（赤）を押すと：
    直前の判定結果を即座に消去。
    主審・塁審ともにオレンジ色にフラッシュし、「操作を取り消しました」と表示。
    カウントが正しい状態に巻き戻ります。

#⑤ 判定リクエスト（塁審がYesを押した場合）
- 塁審が判定に異議がある際、「判定リクエスト」ボタン（青）を押すと：
    主審・塁審ともに青色にフラッシュし、「判定リクエストがありました」と表示。
    審判団は試合を止め、協議に入ります。
